# scenario-test

[English](README.md) | [中文](README.zh.md)

**極めて汎用的な自然言語**でテストケースを記述・実行、コード不要

```gherkin
Scenario: 新規ユーザーが登録を完了する
    Given ユーザーが登録ページにいる
    When すべての必須情報を入力し、登録フォームを送信する
    Then 登録が成功し、ユーザーセンターにリダイレクトされ、入力内容と一致する
```

## インストール

```bash
npx -y agent-add \
  --skill https://github.com/pea3nut/scenario-test.git#skills/scenario-test \
  --command https://github.com/pea3nut/scenario-test.git#commands/scenario-exec.md \
  --sub-agent https://github.com/pea3nut/scenario-test.git#agents/scenario-case-runner.md
```

scenario-test は skill、command、sub-agent のサポートに依存しています。使用している AI コーディングツールがこれらをサポートしていることを確認してください。

## テストケースの生成

インストール後、AI は scenario-test の skill を自動的に認識します。テストしたい内容を自然言語で記述するだけです。

シナリオ1：新機能開発中

```md
/speckit.specify xxx 機能を開発、テスト部分は scenario-test を使用
```

シナリオ2：既存機能のケース作成

```md
すべてのログイン関連 API に対して scenario-test のケースを生成してください。機能カバレッジを完全にしてください
```

## 実行設定

初回のケース生成時、AI が自動的に `scenario-run-config.md` を作成し、プロジェクトコンテキストに基づいて設定を推定します。実行前に確認・修正できます。

| 設定項目        | デフォルト | 説明                                     |
| --------------- | ---------- | ---------------------------------------- |
| `concurrency`   | 4          | 最大同時実行サブエージェント数           |
| `maxFailures`   | 10         | この数の失敗後に早期終了                 |
| `maxRetries`    | 1          | スマートリトライ回数（0 = リトライなし） |

## テストケースの実行

最も重要なのは**エグゼキュータ設定**です。scenario-test はテストケースの実行方法を知る必要があります。例えば、自然言語のケースを curl ベースの API テストや Playwright CLI ベースの E2E テストに変換します。

一般的に、scenario-test はプロジェクトの背景に基づいてエグゼキュータを自動的に判断・設定します。

その他の設定項目にはすべてデフォルト値があり、必要に応じて調整してください：

```md
/scenario-exec {テストケースファイルまたはフォルダ} {オプション、実行設定ファイル。デフォルトで自動選択}
```

実行には**デュアルモデルアーキテクチャ**を採用し、品質・コスト・実行速度を最大化します：

- メインモデル（対話中のモデル）がケースの理解、エグゼキュータ命令への変換、失敗原因の分析、リトライの判断を担当
- サブエージェントモデル（メインモデルが起動する複数のサブエージェントタスク）が実際のテストステップ（ブラウザの起動、リクエスト送信、ボタンクリックなど）を実行

そのため、このコマンドの実行には**賢いモデル**（例：Claude Sonnet）を使用する必要があります。サブエージェントは自動的に安価なモデルを使用します。

## 適用シナリオ

Scenario Test は自然言語でケースを記述するため、**機能レベルのテスト**に最適です。非常に汎用的な記述をサポートします：

- すべての必須フィールドを入力する
- GET エンドポイントの戻り値が送信内容と一致することを確認する
- リストページで先ほど作成したレコードを検索する

テストが実装レベル（特定の関数の実装など）に近づくほど、自然言語の汎用化能力は低下し、Scenario Test の優位性も減少します。
